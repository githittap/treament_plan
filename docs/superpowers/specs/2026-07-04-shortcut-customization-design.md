# 단축키 커스터마이즈 설계 (Shortcut Customization)

> 작성일: 2026-07-04 · 대상 파일: `치료계획.html` (단일 파일 HTML 도구)
> 상태: 승인됨 (브레인스토밍 완료, 구현 계획 작성 대기)

## 목표 (Goal)

사용자가 코드를 고치지 않고, 전용 설정 창에서 단축키를 자유롭게 재배치할 수 있게 한다. 설정은 **컴퓨터(브라우저)별로 독립** 저장되어, 직원마다 자기 컴퓨터에서 자기 단축키를 쓸 수 있다.

## 배경 (Why)

- 현재 단축키를 바꾸려면 매번 코드를 수정해야 함 → 번거롭고 배포가 필요.
- 여러 직원이 각자 다른 컴퓨터에서 사용 → 개인별 커스터마이즈 필요.
- 이 도구는 **서버 없는 정적 GitHub Pages 페이지**이므로, 브라우저 `localStorage`가 개인화 저장소로 자연스럽게 맞는다 (로그인·백엔드 불필요).

## 확정된 결정 (승인됨)

| 항목 | 결정 |
|------|------|
| 저장 위치 | 각 컴퓨터 브라우저의 `localStorage` (컴퓨터마다 독립) |
| 개인화 모델 | 1대 = 1명 가정. 안 바꾼 컴퓨터는 코드 기본값 사용 (기본값 = 안전판) |
| 커스텀 범위 | 단일 글자/숫자 키 전부 (모드·치료종류·스티커·펜). **시스템 키 제외** |
| 시스템 키 | `Ctrl+Z/Y/C/V`·`Delete`·`ESC` 는 고정 (설정 목록에 없음) |
| UI 형태 | 헤더 `⌨️ 단축키` 버튼 → 모달 (기존 '조작설명서' 모달과 같은 형태) |
| 재배치 방식 | 배지 클릭 → 원하는 키 한 번 누르면 배정 (녹음식 캡처) |
| 충돌 처리 | 바꿔치기 + 확인. 뺏긴 기능은 '미지정'(빨강) 표시 |
| 반영 시점 | 즉시 (새로고침 불필요). 버튼 위 키 배지도 자동 갱신 |

## 아키텍처 (핵심 작업)

현재 단축키는 **3군데에 흩어져** 있다:
1. `TREAT_SHORTCUTS` 맵 (L1218) — 버튼 배지 **표시용** (치료종류 11개만)
2. 전역 `keydown` 핸들러 (L1435–1602) — **실제 동작**, 키마다 하드코딩 `if` ~24개
3. `CLAUDE.md` 문서 표 — 일부 낡음

이를 **하나의 정본(canonical) 설정표**로 통합한다.

### 데이터 구조

```js
// 액션 정의 (코드 상수) — 기능마다: id, 그룹, 라벨, 기본키, 실행함수
const SHORTCUT_ACTIONS = [
  { id:'mode_select', group:'모드', label:'선택/이동 모드', def:'A', run:()=>{...} },
  { id:'treat_resin', group:'치료종류', label:'레진', def:'F', run:()=>selectTreat('resin') },
  // ... 총 24개
];

// localStorage 저장 형태 (오버라이드만): { actionId: "키" , ... }
// key 이름: 'treatplan_shortcuts_v1'

// 활성 키맵 = 기본값 위에 오버라이드 병합 → { actionId: 활성키 }
function activeKeymap(){ /* def 병합 + localStorage 오버라이드 */ }
```

### 키 입력 처리 (리팩터)

- 시스템 키(`Delete`·`Escape`·`Ctrl+*`) 처리는 **기존 그대로 맨 앞에서** 먼저 처리 (건드리지 않음).
- 그 뒤 24개 하드코딩 `if` 체인을 **단일 조회+디스패치**로 교체:
  ```js
  const key = normalizeKey(e.key);           // 글자는 대문자화, 숫자는 그대로
  const km = activeKeymap();
  const actId = Object.keys(km).find(id => km[id] === key);
  if(actId){ ACTIONS_BY_ID[actId].run(); }
  ```
- 배지 표시(`TREAT_SHORTCUTS` 사용처)도 활성 키맵을 읽도록 바꿔 **자동 갱신**.

> ⚠️ **가장 조심할 부분**: 중앙 키 처리부를 교체하므로 24개 동작이 모두 이전과 동일하게 작동하는지 회귀 검증 필수. 테스트 하네스가 없으므로 Node 로직 추출 테스트 + 브라우저 E2E로 확인한다 (틀니 기능과 동일 방식).

## 설정 창(UI) 명세

- 헤더에 `⌨️ 단축키` 버튼 추가 (기존 🦷 임플란트 기록 버튼 옆).
- 클릭 → 모달 오픈. 내용:
  - 그룹 헤더 4개(**모드 · 치료종류 · 스티커 · 펜/표시**) 아래 각 기능 줄.
  - 각 줄: `기능 이름` + `[현재 키]` 배지. 미지정이면 빨간 `미지정` 배지.
  - 하단: `기본값으로 되돌리기` · `닫기` 버튼.
- **캡처 흐름**: 배지 클릭 → 배지 "키 누르세요…" 하이라이트 → 다음 키 입력을 그 기능에 배정.
  - `ESC` = 캡처 취소.
  - `Delete`/`Backspace` = 그 기능을 '미지정'으로 비움.
  - 글자(a–z)·숫자(0–9)만 허용. 그 외(F1, Space 등)·시스템 예약키는 거부 + 짧은 안내.
- **충돌**: 누른 키를 이미 다른 기능이 쓰면 → `confirm("그 키는 'RCT'가 쓰는 중. 바꿀까요?")` → 예: 새 기능이 키 획득, 기존 기능 '미지정'(빨강).
- 변경 즉시 `localStorage` 저장 + 배지·버전 뱃지 재렌더.

## 범위 밖 (Out of scope)

- 시스템 키(`Ctrl+Z/Y/C/V`·`Delete`·`ESC`) 재배치.
- 한 컴퓨터 안에서 사람별 프로필 분리 (1대=1명 가정으로 불필요).
- 클라우드/기기 간 동기화.
- 조합키(`Ctrl+…`)·다중 키 매핑 (단일 키만).

## 발견 사항 (참고)

- 현재 코드상 **'발치'는 `1`이 아니라 `X` 키** (CLAUDE.md 표와 다름).
- 펜 도구에 문서에 없는 `K`직선·`J`화살표·`U`사각형 키가 존재.
- 새 설정표는 **실제 동작대로** 기본값을 넣는다. 통합의 부수 효과로 문서-코드 불일치가 정리됨. (원장님이 원하면 이후 발치를 `1`로 직접 재배치 가능.)
- 완료 후 `CLAUDE.md` 단축키 표도 실제와 맞게 갱신 필요.

## 성공 기준 (검증 가능)

1. 헤더 `⌨️ 단축키` 클릭 → 모달 열림, 24개 기능이 4그룹으로 나열, 각자 현재 키 배지 표시.
2. '레진' 배지 클릭 → "키 누르세요" → `Y` 누름 → 배지 `Y`로 변경. 모달 닫고 `Y` → 레진 선택됨. `F`는 더 이상 레진 안 됨.
3. 새로고침 후에도 레진=`Y` 유지 (localStorage).
4. 시크릿창(다른 프로필)에서 열면 레진=`F` 기본값 (컴퓨터별 독립 확인).
5. 충돌: 레진에 `V`(RCT 사용중) 지정 → 확인창 → 예 → 레진=`V`, RCT=미지정(빨강). `V` → 레진 선택.
6. `기본값으로 되돌리기` → 전부 원래 키 복원, localStorage 비워짐.
7. 치료종류 버튼 위 배지도 바뀐 키로 표시.
8. 시스템 키(`Ctrl+Z` 등)는 설정 목록에 없고 이전과 동일하게 동작.
