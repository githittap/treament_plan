# 틀니 다이어그램 (Denture Arch Diagram) 설계

작성일: 2026-07-04
대상 파일: `치료계획.html` (단일 HTML, vanilla JS + Canvas)

---

## 1. 목적

현재 틀니는 **노란 글자 라벨**(예: `RPD 지대치 34,44 / sur.cr …`)로만 표시된다. 이를
**치열 인포그래픽(도식)** 으로 바꿔, 아래 항목을 한눈에 보이게 한다.

- 전체틀니(CD) vs 부분틀니(RPD) 여부
- 부분틀니의 **지대치** 위치 (#31~37, #41~47 및 상악 대응)
- **서베이드 크라운(Sur.Cr)** 위치
- **임플란트 서베이드 크라운(I-Sur.Cr)** 위치 (IARPD)
- **IOD 부착장치**(마그네틱/로케이터/바) 위치
- **결손**(틀니로 대체되는 치아) 위치

도식과 기존 글자 라벨을 **둘 다 유지**한다(도식이 라벨을 대체하지 않음).

---

## 2. 현재 상태 (기존 코드)

- `openDenturePopup(e, existing)` — 약 L2373~2482.
  - 팝업: **악궁**(상악/하악) 칩 + **종류**(CD/RPD/IOD/IARPD/기타) 칩 + 종류별 조건부 **텍스트 입력**(지대치, sur.cr 등).
  - 확인 시 `annotations.push({id:idSeq++, type:'text', x, y, text, fontSize})` 로 노란 글자 라벨 1개 생성.
  - 배치: 상악은 상단 `y=naturalH*0.03`, 하악은 하단 `y=naturalH-boxH-naturalH*0.03`, `x=naturalW/2-20`.
- `DENTURE_KINDS` 배열: `{id,label,abbr}` (CD/RPD/IOD/IARPD/etc).
- 틀니 treatType: L914 `{id:'denture', label:'틀니', shape:'denture', color:'#7c3aed', abbr:'틀니'}`.
- 클릭 핸들러: L1233 `if(t.id==='denture'){ … openDenturePopup(e, null); }`.

이 흐름(옵션 A)을 **그대로 확장**한다 — 팝업 안에 미니 치열도를 추가하고, 확인 시 도식 주석을 추가로 생성.

---

## 3. 사용자 흐름 (옵션 A — 팝업 안에서 클릭)

```
틀니 버튼 → 팝업
 ├ 악궁: 상악 / 하악            (기존)
 ├ 종류: CD / RPD / IARPD / IOD / 기타   (기존)
 ├ [IOD 선택 시만] 부착장치: 마그네틱 / 로케이터 / 바   (신규, 1개 선택)
 ├ [신규] 미니 치열도(14치아) — 치아 클릭 순환으로 상태 지정
 └ 확인 → 캔버스 상단(상악 ∩)/하단(하악 ∪)에 도식 + 글자 라벨 배치
```

- 악궁을 바꾸면 미니 치열도의 치식번호/배열(∩/∪)이 그에 맞게 다시 그려진다.
- 종류를 CD로 바꾸면 미니 치열도 14치아가 전부 결손으로 자동 지정되고 클릭이 비활성.
- 종류를 IOD로 바꾸면 부착장치 칩 행이 나타난다.

## 4. 치아 상태 모델 & 색상

치아마다 상태는 4가지 중 하나. **클릭할 때마다 순환**한다.

| 상태 id | 의미 | 표시 | 색 |
|---|---|---|---|
| `nat` | 자연치 = 지대치 | 흰 동그라미 + 치식번호 | 흰 배경, 회색 테두리 (자연치·지대치 구분 안 함) |
| `sur` | 서베이드 크라운 | 동그라미 + `Sur.Cr` 소라벨 | 초록 `#16a34a` (테두리 `#15803d`) |
| `imp` | 임플란트 서베이드 | 동그라미 + `I-Sur.Cr` 소라벨 | 청록 `#0d9488` (테두리 `#0f766e`) |
| `miss` | 결손(틀니로 대체) | 점선 동그라미 + 치식번호 | 투명 배경, 회색 점선 `#9ca3af` |

**순환 순서**: `nat → sur → imp → miss → nat …`

**IOD 예외**: 종류가 IOD이면 `imp`(청록) 치아의 소라벨을 `I-Sur.Cr` 대신 선택한 부착장치
약어로 표기한다 — 마그네틱=`Mag`, 로케이터=`Loc`, 바=`Bar`. (색은 동일 청록)

---

## 5. 종류별 동작

| 종류 | 미니 치열도 | 비고 |
|---|---|---|
| **CD** (완전틀니) | 14치아 전부 `miss` 자동, 클릭 비활성 | 지대치 없음 |
| **RPD** (부분틀니) | 순환으로 `nat`/`sur`/`imp`/`miss` 지정 | 일반 부분틀니 |
| **IARPD** | RPD와 동일, `imp`=I-Sur.Cr 활용 | 임플란트 서베이드 크라운 지대치 |
| **IOD** (오버덴쳐) | 부착장치 칩 1개 선택 + `imp` 위치 지정 | `imp` 소라벨 = Mag/Loc/Bar |
| **기타(etc)** | 도식 없음 | 이번 버전은 기존 글자 라벨만 |

## 6. 캔버스 렌더링

- 새 주석 타입 `type:'denturearch'` 를 도입한다(기존 `text`/`area`/`tooth`/`free`와 별도).
- 상악 도식은 캔버스 **상단**(∩, 앞니가 위), 하악 도식은 **하단**(∪, 앞니가 아래)에 배치.
- 각 치아 = 작은 원 + 치식번호. 상태별 색/점선/소라벨은 §4 규칙을 따른다.
- 동일 확인 동작에서 **기존 노란 글자 라벨(type:'text')도 함께 생성**한다(둘 다 유지).
- 도식과 라벨은 각각 독립 주석이므로 개별 이동/삭제 가능.

### 도식 지오메트리 (목업에서 검증됨)

```js
// 상악: UP=[17,16,15,14,13,12,11,21,22,23,24,25,26,27]
// 하악: LO=[47,46,45,44,43,42,41,31,32,33,34,35,36,37]
n = nums.length;                 // 14
t = i/(n-1);                     // 0..1
x = pad + t*(w - 2*pad);
s = Math.sin(Math.PI*t);
y_up = (h - tooth)*(1 - s) + pad;  // ∩ 앞니 위 (상악)
y_lo = (h - tooth)*s + pad;        // ∪ 앞니 아래 (하악)
```

## 7. 데이터 구조 (초안)

```js
{
  id: idSeq++,
  type: 'denturearch',
  arch: 'up' | 'lo',                 // 악궁
  kind: 'CD'|'RPD'|'IARPD'|'IOD',    // 종류 (기타는 도식 안 만듦)
  attach: null | 'mag' | 'loc' | 'bar', // IOD 부착장치 (그 외 null)
  states: { 34:'sur', 44:'sur', 36:'imp', 45:'miss', … }, // nat이 아닌 치아만 저장
  x, y,                               // 도식 좌상단 (naturalW/H 좌표계)
  scale                               // 도식 크기 배율
}
```

- `states`에 없는 치식번호는 `nat`으로 간주(저장 절약).
- 좌표계는 기존 주석과 동일하게 이미지 자연 크기(naturalW/naturalH) 기준.

## 8. 범위 (이번 버전)

**포함**
- 상·하악 둘 다 도식화.
- CD / RPD / IARPD / IOD 4종 도식화.
- 클릭 순환 방식 상태 지정(§4).
- IOD 부착장치 3종(마그네틱/로케이터/바).
- 도식 + 글자 라벨 둘 다 생성.

**제외(향후)**
- 기타(etc) 종류의 도식화.
- 캔버스 위 도식을 직접 클릭해 편집(옵션 B) — 이번엔 재편집은 삭제 후 재생성.
- 부착장치를 치아별로 다르게 지정(이번엔 틀니 단위 1종).
