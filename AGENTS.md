# AGENTS.md — Codex 리뷰어 지침 (아산정플란트치과 치료계획 도구)

> 이 파일은 Codex가 이 프로젝트를 **자문(advisory) 리뷰어**로 검토할 때 참고합니다.
> Claude Code가 "어떻게 만드는지"는 `CLAUDE.md`에 있으니 여기서 반복하지 않습니다.
> Codex는 여기 적힌 **"무엇을 의심해야 하는지"**에 집중하세요.

---

## 리뷰 원칙

- **차단 금지 (advisory only)**: 여기서 나온 지적은 참고용입니다. push/커밋을 막지 마세요 —
  오탐 하나로 훅을 `--no-verify`로 우회하는 습관이 붙으면 리뷰 자체가 무력화됩니다.
- 리뷰 결과는 "무엇이 의심되는지 + 파일:줄"로 구체적으로. 문제 없으면 짧게 "이상 없음"만.
- `CLAUDE.md`의 Karpathy 원칙(외과적 변경, 단순함 우선)을 어기는 코드도 지적 대상 —
  요청 범위를 벗어난 리팩터링, 불필요한 방어 코드, 과한 추상화 등.

## 이 프로젝트에서 특히 의심해야 할 지점

1. **단일 파일 전역 스코프 충돌** — 빌드/모듈 시스템이 없어 모든 함수·변수가 전역.
   새 기능 추가 시 기존 함수명·변수명과 충돌하는지, 특히 `redraw()`, `renderAnnList()`,
   `renderTreatList()` 계열과 새 코드의 이름 겹침 여부.
2. **캔버스 좌표 수학** — `baseUnit() = min(naturalW, naturalH) / 10` 기준 스케일링.
   이미지 크기가 다른 X-ray를 올렸을 때도 마커 위치·크기가 깨지지 않는지.
3. **undo/redo 무결성** — 새 annotation 추가 시 `redoStack` 초기화 누락 여부.
   `deletedImageState` 스냅샷이 실제 삭제 직전 상태를 정확히 담는지.
4. **이벤트 중복/경쟁** — `_dblClickFired` 같은 디바운스 플래그가 새 팝업/스티커 로직에도
   일관되게 적용되는지. pointerdown/pointerup 순서 의존성.
5. **다중 HTML 파일 간 로직 드리프트** — 기공차트_리메이크장부_서식.html,
   보철프로토콜_진단기.html, 진행판.html처럼 `치료계획.html`에서 복붙해 만든 새 도구들이
   원본 수정 시 따로 놀지 않는지 (공유 로직인데 파일마다 다르게 고쳐진 부분 없는지).
6. **localStorage 키 버전 관리** — `treatplan_shortcuts_v1` 같은 버전 키.
   스키마가 바뀌면 기존 사용자 localStorage와 충돌하는지.
7. **한글 파일명/인코딩** — git·GitHub Pages 배포 경로에서 한글 파일명 깨짐 여부.
8. **난독화 빌드(`build_obfuscate.js`) 대상 오염** — `치료계획.html` 외 파일을 잘못
   대상으로 잡지 않는지 (현재 배포에는 미사용이지만 존재).

## Codex 실행 가이드

- 이 프로젝트는 대부분 단일/소수 HTML 파일 변경이라 `codex review`로 충분.
  100개 파일 넘는 대규모 변경이 생기면 `codex exec`로 전환 (토큰 폭주 방지).
- 검증 목적 reasoning effort는 기본값(high) 권장 — xhigh는 과도해서 긴 검증 흐름에서
  오히려 일관성이 새는 경향이 관찰됨.
